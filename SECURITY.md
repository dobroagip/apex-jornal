## 🛡️ Уровни защиты

### 1. Firestore Security Rules (серверная защита)

```javascript
// Только SUPERADMIN может менять роли
allow update: if isSuperAdmin() && ...

// Редакторы создают только черновики
allow create: if isEditor() &&
              request.resource.data.published == false

// Редакторы редактируют только свои статьи
allow update: if isEditor() &&
              resource.data.authorId == request.auth.uid &&
              request.resource.data.published == resource.data.published
```

### 2. Frontend Guard (защита UI)

```typescript
// Проверка роли перед отображением
if (!isEditor) {
  return <AccessDenied />
}

// Чекбокс публикации только для админов
{isAdmin && (
  <checkbox>Publish Immediately</checkbox>
)}
```

### 3. Rate Limiting

- 3 загрузки изображений за 10 минут (по IP)
- Защита от спама

### 4. XSS Protection

- Санитизация контента через DOMPurify на бэкенде
- Защита от вредоносного кода

### 5. CSRF Protection

- XSRF токены для всех POST запросов
- Защита от подделки запросов

### 6. Audit Logging

- Логирование всех действий админов/редакторов
- Только чтение для superadmin
- Логи нельзя изменить или удалить

## 📋 Workflow публикации

### Редактор создает статью:

```
1. Редактор логинится
2. Открывает админку
3. Видит: "Editor Mode: Your articles will be submitted for moderation"
4. Заполняет форму
5. Нажимает "Submit Article"
6. Статья сохраняется с published: false
7. Уведомление: "Article submitted for moderation!"
```

### Админ модерирует статью:

```
1. Админ видит все черновики
2. Проверяет статью редактора
3. Редактирует при необходимости
4. Меняет published: false → true
5. Статья публикуется
```

### Админ создает статью:

```
1. Админ логинится
2. Открывает админку
3. Видит чекбокс "Publish Immediately"
4. Заполняет форму
5. Ставит галочку "Publish Immediately"
6. Нажимает "Submit Article"
7. Статья публикуется сразу (published: true)
```

## 🚨 Что может пойти не так?

### Сценарий 1: Злонамеренный редактор

**Попытка**: Редактор пытается опубликовать статью напрямую

```javascript
// Frontend: Чекбокс скрыт для редакторов
{isAdmin && <checkbox />}

// Backend: Firestore Rules блокируют
allow create: if request.resource.data.published == false || isAdmin()
```

**Результат**: ❌ Firestore вернет Permission Denied

---

### Сценарий 2: Редактор пытается редактировать чужую статью

**Попытка**: Изменить authorId или редактировать статью другого автора

```javascript
// Firestore Rules проверяют
allow update: if resource.data.authorId == request.auth.uid
```

**Результат**: ❌ Permission Denied

---

### Сценарий 3: Редактор пытается удалить опубликованную статью

**Попытка**: Удалить свою опубликованную статью

```javascript
// Firestore Rules проверяют
allow delete: if resource.data.published == false
```

**Результат**: ❌ Permission Denied

---

### Сценарий 4: Админ пытается назначить себя superadmin

**Попытка**: Изменить свою роль в Firestore

```javascript
// Firestore Rules проверяют
allow update: if isSuperAdmin() // Только superadmin может менять роли
```

**Результат**: ❌ Permission Denied

---

### Сценарий 5: XSS атака через контент статьи

**Попытка**: Вставить `<script>alert('XSS')</script>` в контент

```javascript
// Backend санитизирует контент
const sanitizedContent = DOMPurify.sanitize(content);
```

**Результат**: ✅ Скрипт удален, безопасный HTML сохранен
